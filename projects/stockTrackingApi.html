Stock Tracking API

Node.js, Express, MySQL, Jest, HTML, CSS, Git

This was my final project, completed in the last week at iO Academy. Working in a Scrum team of four, we built a stock management API allowing the user, a warehouse manager, to add, edit, delete and retrieve product data and manage customer orders. As we knew our data would be relational, we chose to use an SQL database. To make the API scalable, we generated a new SKU for each new product, and a unique order number for each new order. We also built functionality allowing the user to add a new order containing multiple products in varying quantities, which automatically updates the stock quantity available. To avoid duplicating data in the database, the API checks whether a customer already exists before adding an order. We used the Controller Services pattern for the architecture, and Jest for unit testing where possible.
