Portfolio

I am currently studying software development at iO Academy, where I have worked on a range of projects as part of a Scrum team. Each project was completed during a four day sprint. We used Git and GitHub for version control, and code reviewed each other’s code at the end of each task and story. Below are my favourites to date!

Academy Portal (PHP with Slim, MySQL, JavaScript)

We worked with two other student development teams on a legacy codebase - the iO Academy portal where academy admins can follow an applicant's journey through application stages. Our Scrum team first worked on fixing two existing bugs, and then built functionality to lock applicant stages to prevent deletion and make the portal safer to use. We also worked on improving the usability of an option select modal, making it clearer to the user where in the process an applicant currently is and what stage is next. The project was built using the PHP framework Slim using the MVC pattern, and used namespacing and autoloading in line with PSR-12. We unit tested with PHPUnit.

Aptitude Test (React)

For this project, we worked together with two other Academy dev teams on the same code base to rebuild a legacy project previously written in TypeScript. To make the application scalable, we created reusable components working in React. We fetched the data for the test using an existing RESTful API and built a dynamic front end to display the questions and answers using the React-Bootstrap component library and SCSS. To pass variables around our app we used useState() and props, and useEffect() to keep an eye on the score. Once the test was submitted the applicant’s score was calculated and sent as a POST request to update the database.

Muppet App (PHP/MySQL)

OOP project following SOLID principles, we built a PHP application for displaying information about a range of muppets on the front end. We used the Hydrator design pattern to interact with data from a MySQL database, and created functionality to search and generate a random muppet. We built error handling and wrote unit tests where possible. We pair programmed, and code reviewed each others’ work on finishing each task, with the Academy trainer code reviewing each story before merging into main. I particularly enjoyed creating the app architecture and working on the search functionality for the project.

Todo App (Node.JS, Express, MongoDB)

To get to know Node.JS, Express and MongoDB, I built a simple todo application with a (stressful) Christmas theme. This app lets the user add, mark complete and delete tasks, as well as add custom “tags” to each task. I used the Controller Services pattern for the backend architecture, and really enjoyed using Node with Express for routing. Though this tiny app certainly doesn’t use MongoDB to its full potential, it was a joy to work with and I hope to use it more in the future. With more time I would’ve liked to refactor the frontend using React (and the UX leaves a lot to be desired!).

Snake Sweeper (JavaScript game)

I worked as part of a Scrum team to build a custom minesweeper-esque JavaScript game. The user was able to select the size grid to play on, as well as how many ‘mines’ to find. We built functionality to validate user input and then programmatically generate the board with randomised hit boxes. Using event handlers, a modal displayed on hit or miss. We built a ‘lives left’ counter and on finishing the game, the user was presented with the opportunity to play again with a new randomised deck.
